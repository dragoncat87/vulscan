| Version | Supported |
|---|---|
| 0.1.0 | Supported |

## Reporting a Vulnerability

Please **do not open a public GitHub issue** for security vulnerabilities.

Instead, report vulnerabilities through one of the following private channels:

- Email: [SECURITY_EMAIL_PLACEHOLDER]
- GitHub private vulnerability reporting:  
  https://github.com/Dragoncat87/vulscan/security/advisories/new

## What to Include

When reporting a vulnerability, please include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Affected version of vulscan
- Any relevant logs, output, or proof-of-concept details

## Response Timeline

We aim to:

- Acknowledge vulnerability reports within **48 hours**
- Patch critical vulnerabilities within **14 days**, where reasonably possible

Timelines may vary depending on complexity, required validation, and release impact.

## Safe Harbour

Good faith security researchers who report vulnerabilities responsibly and avoid causing harm will not face legal action from the vulscan project maintainers.

Please act responsibly by avoiding:

- Data destruction
- Service disruption
- Unauthorized access to third-party systems
- Public disclosure before a fix is available
